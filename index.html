método

Como trabalho

Do problema à recomendação — como estruturo o trabalho analítico na prática.

01

Entendo a decisão que precisa ser tomada

Antes de qualquer dado ou ferramenta: qual pergunta de negócio precisa ser respondida? Qual decisão vai mudar com esse análise?

02

Mapeio as fontes de dados disponíveis

Plataformas de mídia, CRM, banco de dados, planilhas, APIs — identifico o que existe, o que está confiável e o que precisa ser estruturado.

03

Estruturo coleta, tratamento e padronização

ETL, normalização, deduplicação, reconciliação entre fontes — os dados precisam ser confiáveis antes de virarem análise.

04

Crio visualizações e indicadores acionáveis

Dashboards, relatórios, exportações Excel — projetados para quem vai tomar a decisão, não para impressionar tecnicamente.

05

Automatizo o que for recorrente

Se uma tarefa vai se repetir toda semana, ela vira script, pipeline ou automação. Tempo manual é o recurso mais caro de uma operação de dados.

06

Transformo análise em recomendação prática

O entregável final não é um gráfico — é uma resposta: o que fazer, onde cortar, o que escalar, o que investigar mais.

diferenciais

Por que não é o mesmo que o próximo analista

Não analiso mídia isolada

Conecto investimento, leads, lojas, CRM e jornada do usuário. O número da plataforma precisa dialogar com o que chegou no CRM e o que virou venda.

Não dependo só de dashboard

Quando os dados precisam ser estruturados antes de virar análise, eu construo o pipeline. Não terceirizo o problema técnico para depois poder fazer a análise.

Tenho visão de negócio

Traduzo métricas técnicas em recomendações práticas. CPL alto não é um número — é um sinal de onde o funil está quebrando ou onde a segmentação errou.

Base técnica para executar

Python, SQL, APIs, tracking server-side, BI e automação — quando não existe ferramenta pronta, eu construo. Sem depender de engenharia para resolver problemas de dados de marketing.

Entendo os dois extremos

Tanto a operação de mídia — estratégia, criativos, lances, budget — quanto a infraestrutura analítica por trás dela. Isso evita a lacuna entre "quem analisa" e "quem executa".
